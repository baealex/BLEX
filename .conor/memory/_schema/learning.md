## 언제 작성하는가

- 버그를 해결했을 때 (원인 + 해결책)
- 반복될 수 있는 패턴이나 컨벤션을 발견했을 때
- 새로운 기술/도구 사용법을 학습했을 때
- 삽질 후 "다음에는 이렇게 하자"가 명확해졌을 때

## 파일명 규칙

`L-{YYYYMMDD}-{영문slug}.md`

- 예: `L-20250215-react-perf.md`
- slug는 학습 내용을 2~4단어 영문으로 요약

## 작성 규칙

- 하나의 학습 = 하나의 chunk (여러 주제를 섞지 않음)
- 각 필드는 1~2줄로 핵심만 작성
- 전체 10줄 이내
- 파일 경로는 해당 시에만 작성

## 템플릿

```template
---
type: learning
date: YYYY-MM-DD
tags: [태그1, 태그2]
refs: []
---
**발견**: 무엇을 발견/학습했는지 (1줄)
**원인**: 왜 이런 상황이 발생했는지 (1줄)
**해결책**: 어떻게 해결했는지 (1-2줄)
**파일**: 관련 파일 경로 (해당 시)
```

## 예시

```markdown
---
type: learning
date: 2025-02-15
tags: [performance, react]
refs: []
---
**발견**: 리스트 렌더링 시 500ms 이상 지연 발생
**원인**: key에 index를 사용하여 불필요한 re-render 발생
**해결책**: 고유 ID를 key로 변경 + React.memo 적용으로 50ms로 개선
**파일**: src/components/UserList.tsx
```
