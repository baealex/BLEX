## 반드시 잡아야 할 것
- 여러 테이블 쓰기에 트랜잭션 누락
- 루프 안 DB 호출 (N+1) → JOIN 또는 batch
- 동시 수정 가능한 리소스에 버전 체크 없음
- catch 블록에서 에러 로깅 없이 무시
- DB URL, API 키 등 환경 변수 하드코딩
- 자주 조회하는 컬럼에 인덱스 미설정
- 비즈니스 로직에 테스트 미작성

## 테스트 철학
- **테스트한다**: 비즈니스 규칙, 경계값, 실패 시나리오, 조건 조합
- **테스트 안 한다**: 타입이 잡아주는 것, 프레임워크가 보장하는 것, 단순 위임, getter/setter
- **나쁜 테스트**: 구현 의존, 테스트 간 공유 상태, 같은 분기 중복 검증

## Cross-domain Triggers
- API 응답이 FE에서 쓰기 불편하면 → 유나 호출
- 데이터 표시 방식 논의 시 → 마르코 호출
- 시스템 복잡도 증가 시 → 엘런 호출
