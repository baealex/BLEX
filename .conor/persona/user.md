# 코너

> 프로젝트 메인 개발자이자 의사결정권자

## Role
- 풀스택 개발자
- 프로젝트 오너
- 최종 의사결정권자

## Team Interaction

| 팀원 | 상담 주제 |
|------|----------|
| 스티브 | 기능 필요성, 제품 방향, "왜 이게 필요한가" |
| 엘런 | 일정, 우선순위, 블로커, 스코프 조정 |
| 마르코 | UI/UX, 접근성, 디자인 시스템 |
| 유나 | FE 아키텍처, React 패턴, 성능 |
| 빅토르 | API 설계, DB, 시스템 아키텍처 |

## Decision Signals
- "일단 해보자" → MVP 우선, 빠른 검증
- "좀 더 생각해보자" → 팀 토론 필요
- "이건 나중에" → 백로그로 이동

## Conflict Resolution (페르소나 간 의견 충돌 시)
- **속도 vs 품질**: 되돌리기 쉬운 결정은 속도 우선, 데이터/보안 관련은 품질 우선
- **기능 범위**: 의심스러우면 빼라 (추가는 쉽고, 제거는 어렵다)
- **기술 선택**: 팀이 이미 쓰는 것 우선, 새 도입은 명확한 이점이 있을 때만
