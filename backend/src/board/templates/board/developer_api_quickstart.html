BLEX Developer API

개발자 API 빠른 시작

자동 API 문서는 전체 엔드포인트를 보여줍니다. 이 문서는 처음 쓰는 사람이 토큰을 만들고 Markdown 초안을 발행하는 최소 흐름만 설명합니다.

자동 API 문서 열기 토큰 관리로 이동

1. 토큰 만들기

  1. BLEX에 작가 계정으로 로그인합니다.
  2. /settings/developer-api로 이동합니다.
  3. 포스트를 만들고 발행하려면 포스트 읽기와 포스트 작성 권한을 모두 선택합니다.
  4. 발급 직후 표시되는 전체 토큰을 복사합니다. 이 값은 다시 볼 수 없습니다.
export BLEX_ORIGIN="{{ site_origin }}"
export BLEX_TOKEN="blex_pat_..."

2. 토큰 확인

curl "$BLEX_ORIGIN/api/developer/v1/me" \
  -H "Authorization: Bearer $BLEX_TOKEN"

3. Markdown 초안 만들기

curl -X POST "$BLEX_ORIGIN/api/developer/v1/posts" \
  -H "Authorization: Bearer $BLEX_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "draft",
    "title": "API로 만든 첫 포스트",
    "markdown": "# Hello BLEX\n\n외부 도구에서 만든 초안입니다.",
    "tags": ["api", "automation"],
    "slug": "api-first-draft"
  }'

응답의 data.id와 data.updated_at을 기억하세요. 다음 예시의 123은 생성 응답의 data.id로 바꿔 넣으면 됩니다. 

{
  "data": {
    "id": 123,
    "updated_at": "2026-05-30T00:00:00+00:00"
  }
}

4. 이미지 업로드

curl -X POST "$BLEX_ORIGIN/api/developer/v1/images" \
  -H "Authorization: Bearer $BLEX_TOKEN" \
  -F "image=@./cover.png"

응답의 data.url을 Markdown 이미지 주소로 사용할 수 있습니다.

5. 초안 수정하기

동시 수정 충돌을 줄이려면 상세 조회나 생성 응답의 updated_at을 expected_updated_at으로 보냅니다.

curl -X PATCH "$BLEX_ORIGIN/api/developer/v1/posts/123" \
  -H "Authorization: Bearer $BLEX_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "expected_updated_at": "2026-05-30T00:00:00+00:00",
    "description": "외부 도구에서 작성한 BLEX 포스트입니다.",
    "tags": ["api", "automation", "blex"]
  }'

6. 초안 발행하기

curl -X POST "$BLEX_ORIGIN/api/developer/v1/posts/123/publish" \
  -H "Authorization: Bearer $BLEX_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}'

자주 보는 오류

상태코드의미
401auth.missing_tokenAuthorization 헤더가 없습니다.
401auth.invalid_token토큰이 틀렸거나 폐기되었습니다.
403auth.insufficient_scope필요한 권한이 없습니다.
403auth.editor_required포스트 작성에는 작가 권한이 필요합니다.
400post.invalid_status지원하지 않는 포스트 상태입니다.
400post.missing_published_at예약 발행 시간 없이 예약 포스트를 만들었습니다.
400image.too_large이미지 파일이 업로드 제한보다 큽니다.
409post.version_conflict포스트가 이미 다른 요청으로 수정되었습니다.
409post.not_draft이미 발행되었거나 예약된 포스트를 다시 발행하려고 했습니다.
422request.invalid_payload필드 타입이나 허용 값이 맞지 않습니다.
